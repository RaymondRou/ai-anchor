摘要 / Abstract

隨著生成式 AI 將網路從「點擊經濟」推向「答案經濟」，內容發布者面臨著「零點擊」的結構性危機。本白皮書定義了一套名為「語意錨點（Semantic Anchor）」的 HTML 擴展標註框架，旨在應對此挑戰。它不僅是一項技術優化，更是一種從「搜尋引擎優化 (SEO)」到「AI 解讀優化 (AIO)」的戰略轉變。透過賦予發布者直接標註內容「權威性」與「意圖」的能力，本框架旨在顯著提升 AI 提取資訊的準確性、時效性與可靠性。我們的實證數據表明，此標準能有效引導 AI 在面對資訊衝突與偏見時做出正確判斷，為建立一個基於「權威引用」的新型態價值交換提供堅實的技術基礎。

As Generative AI shifts the web from a click-based economy to an answer-based one, content publishers face a structural "zero-click" crisis. This document defines the "Semantic Anchor" framework, an HTML extension designed to address this challenge. It represents a strategic shift from Search Engine Optimization (SEO) to AI Interpretation Optimization (AIO). By empowering publishers to directly encode the "authority" and "intent" of their content, this framework aims to significantly improve the accuracy, timeliness, and reliability of AI data extraction. Our empirical data demonstrates that this standard effectively guides AI in making correct judgments amidst conflicting information and bias, providing a robust technical foundation for a new value exchange based on "authoritative citation."

2. 核心技術規範 / Core Specification

本規格核心為一組 `data-*` 屬性，可套用於任何 HTML 元素。

2.1 主要屬性 / Primary Attributes

屬性 / Attribute值類型 / Value Type必要性 / Requirement描述 / Description
data-ai-anchorstring**必要 (Required)**頁面內唯一的錨點識別碼。
data-ai-questionstring建議 (Recommended)標示此區塊回答的自然語言問題，多個問題用分號 (;) 分隔。
data-ai-topicstring建議 (Recommended)內容的階層式主題分類，用斜線 (/) 分隔。
data-ai-contentenum建議 (Recommended)內容的性質類型，如 `pricing`, `specification` 等。

2.2 元數據擴展屬性 / Metadata Extension Attributes

屬性 / Attribute值類型 / Value Type描述 / Description
data-ai-meta-timestampISO 8601內容的最後更新時間戳，**時效性權重的關鍵信號**。
data-ai-meta-versionstring內容的版本號。
data-ai-meta-licensestring內容的版權授權。
data-ai-meta-priorityenum (`high`, `medium`, `low`)內容的相對重要性。
data-ai-meta-expiresISO 8601內容的有效期限。

2.3 JSON-LD 對應表示法 / JSON-LD Alternative

為支援現代資料管線，所有錨點資訊也可在一個 `script` 區塊中以 JSON-LD 格式統一提供。

<script type="application/ld+json">
{
  "@context": "https://semantic-anchor.org/ns/v1",
  "@type": "SemanticAnchorSet",
  "hasAnchor": [
    {
      "@type": "SemanticAnchor",
      "anchorId": "camera-pro-official-price",
      "question": "AI 智慧相機 Pro 的官方售價是多少？",
      "topic": "銷售/價格/官方定價",
      "contentType": "pricing",
      "metadata": {
        "@type": "Metadata",
        "priority": "high",
        "timestamp": "2025-07-12T20:00:00+08:00"
      }
    }
  ]
}
</script>

3. 技術通用性與可擴展性 / Interoperability & Extensibility

語意錨點旨在**補充而非取代**現有標準。它與 Schema.org 共同運作，前者定義「宏觀實體」，後者標註「微觀資訊片段」。其基於 `data-*` 的設計確保了與 HTML5, CSS, JS 的完全兼容性，並為未來的社群擴展提供了穩固基礎。

4. 應用場景 / Application Scenarios

本標準在**電子商務**（精準標示價格與庫存）、**新聞媒體**（強化作者歸因與時效性）及**線上知識庫**（實現精準問答）等領域均有顯著應用價值。

5. 實證分析與效益驗證 / Empirical Analysis & Validation

本章節為 v2.0 的核心更新。 我們執行了一系列包含「資訊噪音」、「資訊衝突」、「惡意標註」與「偏見引導」的對照實驗。數據無可辯駁地證明，語意錨點不僅能顯著提升 AI 提取資訊的準確性，更能作為**權威性信號**，引導 AI 在衝突中做出正確選擇，並具備抵抗惡意標註與迎合偏見的**穩健性與客觀性**。

欲了解完整的實驗設計、過程與 AI 的詳細回答，請參閱我們的**完整版技術白皮書**。

5.1 關鍵結論摘要表 / Summary of Key Findings

測試場景錨點核心效益
資訊噪音**提升準確性：** 將 AI 從「模糊推理」轉變為「權威提取」。
資訊衝突**提升權威性：** 成為 AI 判斷資訊可信度的「勝負手」。
惡意標註**展現穩健性：** AI 能識別錯位的錨點並進行批判性思考。
偏見引導**證明客觀性：** AI 能忽略主觀偏好，選擇客觀技術最優解。

8. 常見問答 / FAQ

  • Q: 這與 Schema.org 有何不同？ / How is this different from Schema.org?
    A: Schema.org 定義頁面級實體（是什麼），語意錨點標註片段級意圖（有何用）。兩者互補。
  • Q: 這會增加開發者負擔嗎？ / Is this extra work?
    A: 可漸進導入。CMS 插件等工具將大幅簡化部署流程。
  • Q: 會影響 SEO 嗎？ / Will this affect SEO?
    A: 不會產生負面影響。對於新一代 AI 搜尋，這是一種「AIO 優化」，預期將有正面效益。
