摘要 / Abstract

本文件定義「AI Anchor」規格，一組 HTML 屬性與慣例，旨在提升網頁內容對人工智慧（AI）模型的可機器解讀性。透過明確標記關鍵內容區塊、主題及其回答的用戶問題，AI Anchor 能讓資料擷取更精確、高效且具脈絡。規格亦提出 sitemap-ai.xml 及 JSON-LD 替代表示法，確保廣泛相容與推廣。

This document defines the "AI Anchor" specification, a set of HTML attributes and conventions designed to improve the machine readability of web content for Artificial Intelligence (AI) models. By providing explicit, granular markers for key content blocks, their associated topics, and the user questions they answer, AI Anchors enable more accurate, efficient, and context-aware data extraction. This specification also proposes extensions for content discovery via a specialized sitemap (sitemap-ai.xml) and an alternative representation using JSON-LD, ensuring broad compatibility and adoption.

1. 簡介 / Introduction

1.1 動機：為何需要 AI Anchors？ / Motivation: Why AI Anchors?

現代 AI 模型（特別是大型語言模型）需理解與摘要網頁，但標準 HTML 缺乏明確標記內容脈絡的機制，導致 AI 需依賴推測，易產生誤判或雜訊。AI Anchor 標準彌補此缺口，讓內容創作者能明確標示重點、主題、回答的問題與相關中繼資料，提升 AI 理解與資料品質。

Modern AI models, particularly Large Language Models (LLMs), are increasingly tasked with crawling, understanding, and summarizing the web. However, standard HTML, while excellent for presentation, lacks a native mechanism to explicitly denote the purpose and context of specific content fragments. This ambiguity forces AI models to rely on heuristics to infer which parts of a page answer a specific question or relate to a certain topic, often leading to inaccuracies, hallucinations, or extraction of irrelevant boilerplate content. The AI Anchor standard addresses this gap. It provides a simple, semantic layer upon HTML to:

  • Pinpoint Key Information: Directly flag the most important content sections for an AI.
  • Provide Context: Link content directly to the natural language questions it answers.
  • Improve Classification: Assign structured topic categories to content blocks.
  • Enhance Data Reliability: Attach metadata such as timestamps, versions, and licenses.
By adopting AI Anchors, content creators gain explicit control over how their information is interpreted by machines, leading to better representation in AI-driven search results, chatbots, and data analysis tools.

1.2 設計原則 / Design Principles

  • 簡單與漸進：易於採用，無需重構整站。
    Simplicity & Incrementality: Easy to adopt without requiring a full site rewrite.
  • 互通性：補足而非取代現有標準（如 Schema.org、ARIA）。
    Interoperability: Complements, rather than competes with, existing standards like Schema.org and ARIA.
  • 低負擔：採用標準 data-* 屬性，避免衝突並維持效能。
    Low Overhead: Utilizes standard data-* attributes to avoid conflicts and maintain performance.
  • 可擴展性：核心屬性設計明確，易於未來擴充。
    Extensibility: Designed with a core set of attributes and a clear path for future extensions.

2. 核心規格：HTML 屬性 / Core Specification: HTML Attributes

本規格核心為一組 data-* 屬性，可套用於任何 HTML 元素（如 <section>、<div>、<article>）。
The core of the specification is a set of data-* attributes that can be applied to any HTML element (e.g., <section>, <div>, <article>).

2.3 使用範例 / Usage Example

<article
  data-ai-anchor="service-pricing-2025"
  data-ai-question="你們的企業方案價格是多少？;What is the enterprise plan pricing?"
  data-ai-topic="服務/方案/企業版"
  data-ai-content="pricing"
  data-ai-meta-version="3.0"
  data-ai-meta-timestamp="2025-07-11T10:00:00Z"
  data-ai-meta-priority="high">
  <h3>企業版方案</h3>
  <p>聯繫我們的銷售團隊以獲取客製化報價。</p>
</article>

3. 替代表示法：JSON-LD / Alternative Representations: JSON-LD

為支援語意網與現代資料管線，AI Anchor 可用 JSON-LD 於 <script type="application/ld+json"> 區塊表示，方便爬蟲一次擷取所有 anchor 資料。

To support modern data pipelines and semantic web technologies, AI Anchors can be represented within a <script type="application/ld+json"> block. This allows crawlers to gather all anchor data in a single, structured format without parsing the entire DOM.

3.1 JSON-LD Context

{
  "@context": {
    "ai": "https://your-github-repo/ai-anchor/ns/v1#",
    "AIAnchorSet": "ai:AIAnchorSet",
    "hasAnchor": "ai:hasAnchor",
    "anchorId": "ai:anchorId",
    "question": "ai:question",
    "topic": "ai:topic",
    "contentType": "ai:contentType",
    "version": "ai:version",
    "timestamp": "ai:timestamp",
    "license": "ai:license",
    "priority": "ai:priority",
    "expires": "ai:expires",
    "dependency": "ai:dependency"
  }
}

3.2 JSON-LD 範例 / JSON-LD Example

<script type="application/ld+json">
{
  "@context": "https://your-github-repo/ai-anchor/ns/v1#",
  "@type": "AIAnchorSet",
  "hasAnchor": [
    {
      "@type": "AIAnchor",
      "anchorId": "service-pricing-2025",
      "question": [
        "你們的企業方案價格是多少？",
        "What is the enterprise plan pricing?"
      ],
      "topic": "服務/方案/企業版",
      "contentType": "pricing",
      "version": "3.0",
      "timestamp": "2025-07-11T10:00:00Z",
      "priority": "high"
    },
    {
      "@type": "AIAnchor",
      "anchorId": "faq-refund",
      "question": "退款政策是什麼？",
      "topic": "政策/退款",
      "contentType": "faq"
    }
  ]
}
</script>

5. 發現機制：sitemap-ai.xml 協議 / Discovery Mechanism: The sitemap-ai.xml Protocol

為利爬蟲發現含 AI Anchor 的頁面，建議於網站根目錄放置 sitemap-ai.xml，並於 robots.txt 加註：

To allow crawlers to efficiently discover pages with AI Anchors and the anchors themselves, we propose an extension to the sitemap protocol. A new file, sitemap-ai.xml, should be placed in the root directory and referenced in robots.txt.

Sitemap: https://www.example.com/sitemap-ai.xml

5.1 schema 定義 / Schema Definition

sitemap-ai.xml 採用標準 sitemap XML schema，並於每個 <url> 區塊內可有多個 <ai:anchor> 元素，描述該頁的 AI Anchor。

The sitemap-ai.xml uses the standard sitemap XML schema but introduces a new <ai:anchor> element within each <url> block.

<urlset>: 根元素 / The root element.
<url>: 單一頁面資訊 / Contains information about a single URL.
<loc>: 頁面網址 / The URL of the page.
<lastmod>: （選填）頁面修改時間 / (Optional) The modification time of the page.
<ai:anchor>: （可多個）描述單一 AI Anchor / (Optional, multiple allowed) Describes a single AI Anchor on the page.
<ai:id>: data-ai-anchor 識別碼 / The data-ai-anchor identifier.
<ai:topic>: data-ai-topic 值 / The data-ai-topic value.
<ai:updated>: data-ai-meta-timestamp 值 / The data-ai-meta-timestamp value.

5.2 sitemap-ai.xml 範例 / Example sitemap-ai.xml

<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:ai="https://your-github-repo/ai-anchor/ns/v1/sitemap">
  <url>
    <loc>https://www.example.com/services/enterprise</loc>
    <lastmod>2025-07-11T10:00:00Z</lastmod>
    <ai:anchor>
      <ai:id>service-pricing-2025</ai:id>
      <ai:topic>服務/方案/企業版</ai:topic>
      <ai:updated>2025-07-11T10:00:00Z</ai:updated>
    </ai:anchor>
    <ai:anchor>
      <ai:id>contact-sales</ai:id>
      <ai:topic>聯絡資訊/銷售</ai:topic>
      <ai:updated>2025-01-05T00:00:00Z</ai:updated>
    </ai:anchor>
  </url>
  <url>
    <loc>https://www.example.com/support/faq</loc>
    <lastmod>2025-06-20T14:00:00Z</lastmod>
    <ai:anchor>
      <ai:id>faq-refund</ai:id>
      <ai:topic>政策/退款</ai:topic>
      <ai:updated>2025-06-20T14:00:00Z</ai:updated>
    </ai:anchor>
  </url>
</urlset>

6. 常見問答 / Frequently Asked Questions (FAQ)

  • Q1: 這與 Schema.org 有何不同？
    How is this different from Schema.org?
    A: Schema.org 適合標註整頁或主要實體，AI Anchor 更細緻，標記回答特定問題的片段。兩者可互補。
    A: Schema.org is excellent for defining the semantics of an entire page or major entities on it (e.g., this is a Product, this is a Recipe). AI Anchors are more granular. They pinpoint the specific fragment of content that answers a specific question. They are complementary: you can have Schema.org's Product markup on your page, and use AI Anchors to flag the exact <div> that contains the specification or answers a pricing question.
  • Q2: 會增加開發者負擔嗎？
    Is this a lot of extra work for developers?
    A: 可漸進導入，先標記重要內容。WordPress 外掛等工具可自動化。
    A: It can be implemented incrementally. Start by marking up only your most critical content on high-traffic pages. The proposed WordPress plugin and other CMS integrations aim to automate much of this process, making it as simple as filling out a few fields.
  • Q3: 會影響 SEO 嗎？
    Will this negatively impact my SEO?
    A: 不會。data-* 屬性為合法 HTML5，無理解者會忽略，有理解者（如 AI 搜尋）則可提升曝光。
    A: No. The use of data-* attributes is valid HTML5 and is ignored by crawlers that do not understand it. For crawlers that do understand it (especially future AI-driven search engines), it is expected to have a significant positive impact by providing clearer signals about your content's relevance and structure, potentially improving your chances of appearing in featured snippets and AI-generated answers.
  • Q4: 可以套用在任何 HTML 元素嗎？
    Can I use AI Anchors on any HTML element?
    A: 可以，區塊元素（如 section、article、div）最常見，亦可用於小型內容（如 span）。
    A: Yes. You can apply AI Anchor attributes to block-level elements like <section>, <article>, <div>, or even inline elements like <span> if the target content is very small, though block-level elements are the more common use case.
  • Q5: 沒填所有屬性會怎樣？
    What happens if I don't provide all the attributes?
    A: 只有 data-ai-anchor 為必填，其他屬性建議補充，愈多愈利於 AI 理解。
    A: Only data-ai-anchor is strictly required to define an anchor. All other attributes are recommended for providing richer context. The more attributes you provide, the more effectively an AI can understand and utilize your content.
